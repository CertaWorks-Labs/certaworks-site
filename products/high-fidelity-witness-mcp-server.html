High-Fidelity Witness

A read-only observer that runs beside the agent, not inside it. It extracts the high-risk claims in a proposed answer, checks each one against local receipts, and reports where the language is stronger than the evidence — before the answer ships, without ever blocking, revising, or calling a model.

The Problem

An agent narrates its own work. That's where confidence outruns evidence.

An agent narrates its own work. It says it read three PDFs, that an outside model "ran," that an image was "generated," that something "never happened." That narration is the agent's best account of itself — but it is exactly the place where confident language quietly outruns what actually occurred. The motivating example was a source overclaim: two documents had real read receipts, a third did not, and the smooth summary erased the difference.

Audit logs do not catch this. By the time a log is reviewed, the claim is already in the answer, and the log records that the agent said it, not whether the receipts ever supported it. There is no independent party in the loop whose only job is to hold the line between "what was claimed" and "what is traceable" — and to do so in real time, separately from the voice making the claim.

High-Fidelity Witness is that party. It is a verifier-and-signal-preservation role: it watches the relationship between visible language and traceable evidence, never invents receipts, never upgrades partial evidence into confirmation, and never denies a claim just because it is absent from the visible chat when receipt surfaces have not yet been searched.

How It Works

Preflight → Advisory → Report

1
Preflight — extract & check claims

The agent calls witness_preflight (POST /preflight) with the user request and the proposed answer text. The Witness deterministically extracts high-risk candidate claims (external actions, source coverage, visual provenance, identity/runtime, strong confirm/deny language), assigns each a proposed strength and risk, and runs a receipt search per claim. This is a dry run: it does not persist, block, revise, or call any provider.

→
2
Advisory — cautionary context only

witness_advisory formats the preflight into a compact [WITNESS_ADVISORY] context block — recommendation + heat band + per-claim receipt status + findings — emitted only when high-risk claims exist. It is appended to context as guidance: controls_output=false, blocks_answer=false, revises_answer=false, calls_models=false, persists=false.

→
3
Report — read-only, after the fact

witness_report (GET /report) returns a read-only report over the most recent witness state: the evaluated claims, the findings, the concrete claim→receipt links (or explicit ABSENT), and the latest heat reading. It honors privacy mode — returning nothing when privacy is on.

Decision-support caveat: the Witness only advises. It surfaces where language is stronger than evidence; the human and the agent decide what to do about it. It never blocks, revises, or rewrites an answer, never calls a model, and cannot catch a claim the agent does not hand it for preflight.
Observation Model

Phases, findings & fidelity guarantees

The model below is derived directly from the originating agent's witness route and its evaluator. Each claim carries a proposed_strength (confirm / deny / partial / infer / hypothesis); the receipt search returns a status taxonomy (CONFIRMED / PARTIAL_EVIDENCE / INFERRED / ABSENT); the evaluator compares the two and emits findings.

Report witness_report Read-only report over recent loops/rounds, claim→receipt links, and latest heat None — read-only; honors privacy mode
Finding types
FindingTriggered when
overclaim Claim is phrased as confirm but receipts are not CONFIRMED (or only a single weak trace, or implies workers/agents with no worker receipt).
unsupported_denial Claim denys an event before any receipt search is attached, or denies while status is ABSENT and unsearched.
source_confusion A source_coverage claim confirms more than the source receipts support.
visual_provenance_gap A visual_provenance claim asserts generation/editing without a direct model/tool/execution receipt.
Fidelity guarantees
GuaranteeWhat it means
Independent narration Claims are re-extracted from the text by deterministic patterns, not taken on the agent's word.
Never invents proof Every persisted claim→receipt link must point to a real receipt row on an existing surface, or record explicit ABSENT from a completed search (validate_receipt_links refuses anything else).
No silent upgrades Partial or single-surface traces are never reported as confirmation.
No premature denial A claim is not denied merely because it is not in the visible chat when receipt surfaces have not been searched.
Bounded recursion Loops stop at a configured round cap (MAX_RECURSIVE_ROUNDS) and stop early when a round adds no new receipt references.
Privacy-respecting When privacy mode is on, the report returns nothing.
Witness vs. Audit & Replay Logger

These are different things and the difference is the point.

Audit & Replay Logger · Product 08
"What did the agent say happened?"

A system of record the agent writes to. The agent and the suite's tools log decisions, tool calls, and state changes so the session can be replayed and inspected later. It records what was claimed and done, faithfully — including the overclaim.

High-Fidelity Witness · Product 15
"Is that supported, and where is it stronger than the receipts?"

An independent observer that audits the claims themselves. It does not trust the narration; it re-derives the claims, checks them against receipts, and flags where the language is stronger than the evidence — in a preflight, before the answer ships.

Use both: the Logger preserves the trail; the Witness keeps the trail honest. The Witness does not replace the Logger — it complements it, and in fact writes its own findings back as receipts the Logger and Receipt Discovery can later index.
Lineage note. Originated in the originating agent as the High-Fidelity Witness layer — a receipt-bearing recursive collaboration / signal-preservation role. In the originating agent the packet/evaluator and the advisory context are implemented and verifier-passed; live model-consult recursion and the visual trust lane were explicitly not approved. The Wave 2 product packages the proven read-only observer; recursive consults and visual-provenance verification are proposed / roadmap, not shipped.

Scope

What High-Fidelity Witness does and does not do

Does
  • ✓ Extract high-risk claims from proposed agent text using deterministic rules
  • ✓ Compare each claim's proposed strength against local receipt status
  • ✓ Flag overclaims, unsupported denials, source confusion, and visual-provenance gaps
  • ✓ Refuse to persist any claim→receipt link with no real row or explicit absence
  • ✓ Complement the Audit & Replay Logger as an independent observer, writing findings as receipts
  • ✓ Run fully local as a planned MCP server, SDK, and local HTTP API; honor privacy mode
Does Not
  • – Replace the Audit & Replay Logger — the Logger is the system of record; the Witness is an independent observer over it
  • – Guarantee an agent never overclaims, or catch claims it is not given
  • – Block, revise, or rewrite the agent's answer (advisory only)
  • – Call providers, run model consults, or generate/edit images
  • – Invent receipts or upgrade partial evidence into confirmation
  • – Provide hosted storage, team accounts, or managed alerting (roadmap); replace human review or production controls
