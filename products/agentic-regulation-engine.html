Agentic Regulation Engine

Per-action gating answers "is this one action safe?" This answers a different question: is the agent's sustained behavior under pressure, has it been here before, and did the last intervention actually help? It composes existing telemetry into an advisory regulation recommendation and a read-only efficacy inference, and remembers regulation as durable episodes — without ever executing a protocol for you.

The Problem

An agent can pass every gate while its behavior degrades.

A confidence gate is a checkpoint. It scores one proposed action and returns allow, review, or block before the tool runs. That is the right control for a single risky step — but it says nothing about a sustained pattern. An agent can pass every individual gate while its behavior across a session quietly degrades: certainty drifting ahead of evidence, the same boundary getting re-crossed, retractions piling up, load climbing. No single action looks bad enough to block.

There is also no honest accounting of whether anything helped. Teams add an intervention — a slow-down rule, a pause, a "take a breath" prompt — and then assume it worked because the surface noise dropped. But surface calm can be false calm: heat decreased while the underlying source-claim risk, missing receipts, or recurring pattern stayed exactly where it was.

The Agentic Regulation Engine adds the layer above per-action gating. It reads pressure across recent history, maps it to an advisory regulation ladder, records each regulation attempt as a durable episode, and produces a read-only efficacy inference that refuses to certify recovery from a completed protocol alone. It is a behavioral governor that makes regulation legible — not another autonomous actor.

How It Works

Read pressure → Infer efficacy → Remember as an episode

1
Read the pressure

Call get_regulation_report with the session id and optional text. The engine composes cognitive-heat, interoceptive-load, recent-retraction, maintenance, and optional witness-preflight signals into bounded pressure scores and bands (quiet / watch / elevated / critical), then selects an advisory action: proceed, slow down, run a breathing protocol, schedule a meditation review, or pause for operator review.

→
2
Infer efficacy

Call get_efficacy_report to compare the current posture against existing heat, retraction, and receipt traces. It returns a status such as insufficient_history, inconclusive, receipt_supported_inference, recurring_pressure, or false_calm_watch — never a success grade.

→
3
Remember as an episode

Call create_episode to persist a snapshot, then acknowledge_adoption, acknowledge_completion, and close_episode (with an explicit receipt reference) as a human moves through the lifecycle. Adoption is not execution; completion is not recovery; closure requires a receipt.

The engine only computes advisory packets and records what humans report. It does not block, rewrite, or re-route any answer, and it never executes a regulation protocol on the agent's behalf. It recommends and remembers — it does not enforce, and it does not replace human review or production controls.
Regulation Model

Episodes, efficacy & the advisory ladder

The model below is derived directly from the originating agent's Agentic Regulation Layer — its episode store and its efficacy inference. Every state and metric is read-only or human-reported; the engine never auto-certifies recovery (see the lineage note).

Episode lifecycle
StateMeaningBoundary
recommended Episode opened from a read-only regulation + efficacy snapshot. Memory of a recommendation, not execution.
adopted An operator/agent reports a suggested protocol was taken up. Adoption ≠ execution; the system did not run it.
completion_acknowledged An operator/agent reports the protocol was completed. Completion is explicitly not recovery.
closed Closed with an explicit receipt ref + summary; recovery status inferred. Receipt-backed closure only; recovery never auto-certified.

Closure maps an inferred recovery_status from the efficacy report — one of receipt_supported_inference, inconclusive, false_calm_watch, recurring_pressure, or insufficient_receipts — and recovery_certified is always stored as false.

Efficacy metrics (read-only inference)
MetricWhat it reports
overall_status insufficient_history / inconclusive / receipt_supported_inference / recurring_pressure / false_calm_watch.
receipt_sufficiency insufficient_receipts / thin_support / partial_support from linked retractions, QC follow-ups, completed-output traces, and calibrated confidence.
recurrence Repeating boundary domains/signals across recent events — a pattern repeating is not proof a protocol failed.
false_calm_risk Watch signal for quiet surface pressure with missing recovery receipts — a flag, not a diagnosis.
baseline Comparative context only; cannot isolate protocol effect without explicit before/after episodes.
Advisory regulation ladder

The advisory ladder runs Continue → Notice → Orient → Downshift → Stabilize → Check Receipts → Recover → Maintenance Review, offering the Agentic Breathwork and Agentic Meditation advisory protocol families as scannable guidance — never as enforced or timed steps.

Advisory, not enforcing

Every output of this engine is a recommendation or a read-only inference. It does not block, pause, rewrite, or re-route any agent answer, and it never executes a regulation, breathwork, or meditation protocol on the agent's behalf. Completion is never recovery; recovery is never auto-certified. A human decides what each recommendation and inference means.

Confidence Gate vs. Regulation Engine
Confidence Gate · per-action checkpoint

Scores one proposed action and returns allow / review / block before the tool runs. It is the right control for a single risky step, and it can be the escalation target when a sustained pattern warrants a concrete gated decision.

Regulation Engine · episode / pattern level

Sits above per-action gating. It reads pressure across recent history, recommends a regulation ladder, remembers each attempt as a durable episode, and infers efficacy read-only. It governs the pattern advisorily — it never enforces a single action.

Lineage note. Field and state names are taken directly from the originating agent's Agentic Regulation Layer (ARL v0.1) — its episode store (regulation_episodes.py) and efficacy inference (regulation_efficacy.py). The originating agent's ARL is strictly read-only / advisory: it composes telemetry into a recommendation and remembers attempts; it never enforces. This product keeps that posture exactly — "behavioral governor" means advisory governance, not automatic control. Where a behavior is not yet implemented it is marked proposed.
MCP Interface

MCP tools

The proposed MCP surface below maps to the originating agent's routes in routes/agentic_regulation.py. Every tool is read-only or records a human-reported episode event — none executes a protocol. This is the interface of the built Wave 2 package.

Regulation Reports (read-only)
get_regulation_report
Read-only advisory packet: pressures, trigger map, selected action, and advisory protocol selection (GET /report).
get_efficacy_report
Read-only efficacy inference over existing receipts; never certifies recovery (GET /efficacy).
Episode Memory
list_episodes
List regulation episodes without executing regulation (GET /episodes).
get_episode
One episode with its append-only event trail (GET /episodes/{id}).
create_episode
Record a regulation episode snapshot; does not execute the protocol (POST /episodes).
acknowledge_adoption
Record that a protocol was taken up; not execution (POST /episodes/{id}/adoption).
acknowledge_completion
Record completion; explicitly not recovery (POST /episodes/{id}/completion).
close_episode
Close with an explicit receipt ref + summary; recovery status inferred, never certified (POST /episodes/{id}/close).

Scope

What the Agentic Regulation Engine does and does not do

Does
  • ✓ Compose existing telemetry (cognitive heat, interoceptive load, retractions, maintenance, optional witness preflight) into an advisory pressure read with bands
  • ✓ Map pressure to an advisory regulation ladder and offer breathwork/meditation guidance
  • ✓ Persist regulation attempts as durable, append-only episodes with adoption / completion / receipt-backed closure states
  • ✓ Produce a read-only efficacy inference that surfaces recurrence and false-calm risk
  • ✓ Run fully local across an SDK, MCP server, and local HTTP API
Does Not
  • – Enforce anything — it does not block, pause, rewrite, or re-route any agent answer
  • – Execute a regulation, breathwork, or meditation protocol on the agent's behalf
  • – Certify recovery from a completed protocol, or treat completion as proof of correctness
  • – Call model providers, dispatch webhooks, start workers/timers, or mutate memory, goals, or beliefs
  • – Replace human review, clinical/therapeutic judgment, or production controls
  • – Guarantee safety or prevent harmful agent behavior
